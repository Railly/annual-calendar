## 🚀 Getting Started

### Use the v0 Template (Recommended)

[![Use v0 Template](https://img.shields.io/badge/Use%20Template-v0-black?style=for-the-badge&logo=vercel)](V0_TEMPLATE_URL)

### Clone and Run Locally

```bash
# Clone the repository
git clone [GITHUB_URL]

# Navigate to the directory
cd annual-calendar-2026

# Install dependencies
npm install

# Run the development server
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to view the calendar.

## 📖 Usage

### Creating Events
1. Click any day cell to open the event creation modal
2. Fill in title, description, dates, and choose a color
3. Optionally upload an image
4. Click "Create Event"

### Editing Events
- Click on any event bar to edit
- Right-click for quick actions (edit, duplicate, delete)

### Adding Photos & Notes
- Hover over any day cell
- Click the photo icon (📷) to upload a background image
- Click the note icon (📝) to add a journal entry

### Navigation
- Use arrow buttons to navigate years
- Click "Today" to jump to current date (with pulse animation)
- Switch views using the Day/Week/Year toggle

## 🎨 Customization

### Colors
Edit `lib/calendar-data.ts` to modify the event color palette:

```typescript
export const eventColors = [
  { name: "Orange", value: "#f97316", ... },
  // Add your colors here
];
```

### Initial Events
Modify the `initialEvents` array in `lib/calendar-data.ts` to customize default events.

## Vercel Community Forum Blog Post

### Title
**Building a Premium Annual Calendar with v0: A Deep Dive**

### Overview/Introduction

```markdown
# Building a Premium Annual Calendar with v0: A Deep Dive

As a v0 Ambassador, I'm always looking for projects that push the boundaries of AI-assisted development. This time, I challenged myself to build something that would traditionally require significant design and development time: a premium annual calendar application.

The result? A fully-featured calendar built entirely through conversation with v0.

## The Challenge

I started with two reference images of annual calendars — one from a popular email app, another from a project management tool. My prompt was simple:

> "Replicate this annual calendar for 2026, it will be valued in $500"

What followed was a collaborative development session that produced a production-ready application.

## Key Features Built

### 1. Three View Modes

The calendar supports Year, Week, and Day views, each optimized for different planning needs:

- **Year View**: A 21×18 grid displaying all 365 days with continuous event bars spanning multiple cells
- **Week View**: Vertical layout with rotated month labels, perfect for weekly planning
- **Day View**: Hourly breakdown showing all events for a single day

### 2. Continuous Event Bars

One of the most complex features was implementing events that span multiple days as continuous colored bars. This required:

- Calculating event positions across grid cells
- Rendering partial bars at row breaks
- Stacking multiple overlapping events
- Handling edge cases at month boundaries

v0 handled all of this complexity through iterative prompts.

### 3. Rich Interactions

The calendar includes:

- Click-to-create event modals
- Click-to-edit with pre-filled values
- Right-click context menus
- Hover icons for quick actions (photos, notes)
- Touch-friendly mobile drawers

### 4. Visual Polish

Small details that make the difference:

- Diagonal stripe pattern for past days (with `background-attachment: fixed` for seamless alignment)
- Highlighted current day with teal background
- Grayed weekend columns
- Month badges on the first day of each month
- Smooth pulse animation when clicking "Today"

## The Development Process

What made this project special was how natural the iteration felt. Instead of writing code, I provided feedback like:

- "Make the weekends darker"
- "The tooltip arrow should be white"
- "Use this striped background for past days"
- "Move the day label to the top right corner"

v0 understood context, maintained state across a long conversation, and produced clean TypeScript code throughout.

## Tech Stack

- **Next.js 15** with App Router
- **React 19** with latest features
- **Tailwind CSS v4** for styling
- **shadcn/ui** for accessible components
- **Radix UI** primitives under the hood
